# GUI 搜索结果封面渐进加载设计

## 背景与根因

GD Studio 的搜索响应为每首歌曲提供顶层 `pic_id`，但不是可以直接用于 `<img src>` 的封面 URL。项目已有 `get_pic_url()` 通过站点签名接口把 `pic_id` 解析为真实图片 URL；当前仅在下载阶段调用它。GUI 搜索桥接直接返回 `Song.to_result_dict()`，其中有 `pic_id`、没有 `cover`，而结果组件只在 `song.cover` 存在时渲染图片，因此所有搜索结果都显示默认图标。

真实接口验证表明，网易云、Spotify、酷我音乐的 `pic_id` 均可经 `get_pic_url()` 得到 HTTP 200 图片。20 条网易云结果的原搜索约需 1.16 秒，逐首解析封面额外约需 9.98 秒，不能阻塞搜索返回。

## 目标

- GUI 搜索结果保持快速返回，并在后台逐张显示真实封面。
- 继续使用现有签名、Cloudflare 重试和音源适配逻辑解析封面 URL。
- 单张封面失败不影响歌曲列表和其他封面。
- 新搜索启动后停止旧搜索的待处理封面任务，避免陈旧事件污染当前列表。
- 不改变 CLI 搜索性能和输出结构。

## 架构

### GUI Bridge

`MusicBridge.search()` 完成搜索后立即返回现有结果字典，同时启动一个守护线程处理该批歌曲的封面。后台线程按歌曲顺序解析，且每次 Playwright 操作仍通过 `_PlaywrightThread.submit()` 进入专用浏览器线程，保持现有线程安全约束。

`MusicBridge` 增加封面回调。每张封面解析成功后回调以下事件数据：

```text
{
  "id": <歌曲 ID>,
  "source": <音源>,
  "cover": <真实图片 URL>
}
```

没有 `pic_id`、返回空 URL 或解析抛出异常时不发送事件，继续处理下一首。

### 陈旧任务控制

`MusicBridge` 保存递增的封面任务代次。每次 GUI 搜索开始时增加代次，后台任务在每次解析前和发送事件前检查自己的代次；不再是当前代次时立即停止。关闭 bridge 时同样增加代次，使待处理任务退出。

事件以 `id + source` 标识歌曲。即使同一歌曲出现在新搜索中，应用相同封面也是安全的；代次检查负责阻止旧任务继续占用浏览器队列。

### Python GUI API

`MusicApi` 把 bridge 的封面回调转换成 `py-cover` 自定义事件，复用现有 `_emit()` JSON 序列化和 pywebview 事件通道。

### Svelte 前端

前端 API 模块增加 `CoverDetail` 类型和 `onPythonCover()` 监听器。`App.svelte` 在挂载时订阅，在卸载时移除；收到事件后以不可变数组更新当前 `songs` 中 `id`、`source` 都匹配的项，写入 `cover`。

`ResultList.svelte` 已经在 `song.cover` 存在时渲染 `<img>`，无需修改其视觉结构。图片自身加载失败仍遵循浏览器行为，本次不增加重试或本地缓存。

## 错误处理

- 单张解析失败：保留默认图，继续下一张。
- 浏览器会话关闭或提交失败：当前后台任务安静退出。
- 新搜索开始：旧任务停止，不发送后续事件。
- 搜索结果为空：不启动封面线程。
- 不为逐张封面失败写运行日志，避免最多 20 条重复警告。

## 测试设计

- Python bridge 测试：搜索立即返回不带 `cover` 的基础结果，后台逐首发送封面事件。
- Python bridge 测试：某首解析异常后仍继续发送后续歌曲事件。
- Python bridge 测试：代次失效时停止旧任务。
- Python API 测试：封面回调通过 `py-cover` 发给窗口。
- 前端测试：存在封面事件监听、不可变更新和卸载清理。
- 运行 pytest、Ruff、mypy、前端 Node 测试、Svelte check 和 Vite build。
- 用隔离 Chrome 真实搜索三个音源，确认搜索结果先返回且随后收到有效封面 URL；不下载音频。

## 非目标

- 不把封面解析加入 CLI 搜索。
- 不在前端按音源拼接 CDN URL。
- 不并发轰炸站点封面接口。
- 不下载或持久化搜索结果封面到本地。
- 不修改当前结果行尺寸和封面视觉样式。
